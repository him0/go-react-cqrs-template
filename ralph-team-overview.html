Ralph Team
Agent Teams 並列タスク実行

Claude Code の Agent Teams を使い、複数のワーカーエージェントが
worktree で隔離された環境で並列に実装 → テスト → PR作成 → 自動マージまで完結

SCROLL
SECTION 01

アーキテクチャ

リーダーがタスクを読み込み、ワーカーを worktree で並列生成。 各ワーカーが独立して実装から PR 作成まで完結する。 

  /ralph-team
       |
  [Leader] ── タスク読み込み → スケジュール作成 → チーム作成
       |
       |── Batch 1 ──┬── Worker-04 (worktree) → CI → PR → merge
       |              ├── Worker-08 (worktree) → Docker → PR → merge
       |              └── Worker-05 (worktree) → ErrorBoundary → PR → merge
       |
       |── Batch 2 ──┬── Worker-01 (worktree) → Health → PR → merge
       |              └── Worker-07 (worktree) → DarkMode → PR → merge
       |
       |── Batch 3 ...
       |
  [Complete] ── 最終レポート → チームクリーンアップ
SECTION 02

全体ワークフロー

リーダーのスケジューリングからワーカーの自律実行まで。

01

タスク読み込み & スケジュール

/ralph-team でリーダーが起動。タスクファイルを読み込み、競合グループを考慮してバッチスケジュールを作成。

→ TeamCreate でチーム作成
02

チーム作成 & タスク登録

TeamCreate + TaskCreate で各タスクを登録。依存関係（blockedBy）でバッチ順序を制御。

→ concurrency 数だけワーカー生成
03

ワーカー並列実行

各ワーカーは worktree で隔離された環境で自律的に実装。タスクファイルの実装ガイドに従い、検証まで完了。

→ 検証パス後
04

PR 作成 & Auto Merge

ワーカーが gh pr create → gh pr merge --auto --squash で PR を作成し、自動マージを設定。

→ リーダーに報告
05

バッチ進行

バッチ内の全ワーカーが完了 → 次バッチのワーカーを生成。競合するタスクが同時に実行されないことを保証。

→ 全バッチ完了後
06

最終レポート

全タスクの成功/失敗・PR URL をサマリー表示。TeamDelete でクリーンアップ。

SECTION 04

タスク一覧

テンプレートを本番レベルに引き上げる 12 のタスク。 各タスクは独立した PR として完成する。

TASK 01 Backend

ヘルスチェックエンドポイント

/healthz と /readyz を追加。Kubernetes の liveness/readiness probe 対応。

TASK 02 Backend

Graceful Shutdown

SIGINT/SIGTERM シグナルを受信し、処理中リクエストの完了を待って安全に停止。

TASK 03 Security

セキュリティヘッダー

CSP, X-Frame-Options, HSTS 等のセキュリティヘッダーを全レスポンスに付与。

TASK 04 CI/CD

バックエンド CI

GitHub Actions で Go のビルド・lint・vet・テストを PR ごとに自動実行。

TASK 05 Frontend

Error Boundary

React Error Boundary で未キャッチエラーをハンドリング。リトライボタン付き。

TASK 06 Frontend

Toast 通知

sonner ライブラリで操作成功/失敗のフィードバック。既存 CRUD に統合。

TASK 07 Frontend

ダークモード

Tailwind dark class + テーマトグル。システム設定自動追従 & localStorage 保存。

TASK 08 Infra

マルチステージ Dockerfile

フロントエンドビルド + Go バイナリを軽量 Alpine イメージにパッケージ。

TASK 09 Backend

レートリミッター

Token Bucket アルゴリズムで IP 単位のレートリミット。429 + Retry-After 対応。

TASK 10 Backend

リクエスト ID 伝播

リクエスト ID を全レイヤー（handler→usecase→command）の Context で伝播。

TASK 11 Backend

環境別設定管理

envconfig で構造化された設定。dev/staging/prod の分離と .env.example 文書化。

TASK 12 Frontend

フォームライブラリ統合

react-hook-form + zod で既存フォームを移行。スキーマバリデーション統合。

SECTION 05

使い方

タスクの確認から並列実行まで。

STEP 1 — タスク一覧の確認
$ task ralph:list
全タスクの番号・ファイル名・概要を一覧表示。
STEP 2 — タスク内容の確認
$ task ralph:show NUM=01
タスクの全文を表示。要件・実装ガイド・検証手順を確認。
STEP 3 — 実行状況の確認
$ task ralph:status
PR の状態と worktree の状況をタスクごとに表示。
STEP 4 — Ralph Team で並列実行
> /ralph-team
> /ralph-team --tasks 01,02,03
> /ralph-team --concurrency 2
> /ralph-team --dry-run
Claude Code 内で実行。Agent Teams が worktree 隔離された複数ワーカーを
並列生成し、各タスクを独立した PR として自動完成させる。
